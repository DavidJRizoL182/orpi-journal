# ORPI Journal

Daily journal of real decisions made by the Steward of ODS-Foundation, logged in [ODS](https://github.com/ODS-Foundation/ods-specification) format.

## Purpose

This repository is a public eat-your-own-dog-food practice. The Steward of an open decision standard should use that standard to log their own operational decisions. Otherwise the standard is theory without practice.

## Format

Each entry is a JSON file conforming to ODS Core v2 (and where applicable, a community-authored profile). Files are named `YYYY-MM-DD-HHMM-{short-description}.json`.

Some entries are core-only records (governance attestations, project meta-decisions). Others use community profiles (e.g., `ORPI-Operations/v1` once authored) to log operational decisions with structured action vocabularies.

## Why public

Compounding intelligence requires public auditability. Private journals can drift into self-justification. Public journals are a forcing function for honest recording — what is the policy_hash actually? what were the real counterfactuals?

## What this is not

This is not ODS-Foundation governance. It is the personal practice of the Steward using ODS as a tool. Decisions about ODS-Foundation itself (specifications, releases, governance) are recorded in the `ods-specification` repository as RFCs, CHANGELOG entries, and commits. The journal here is operational, not normative.
